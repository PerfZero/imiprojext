# Как смотреть логи в Android Studio

## Открытие Logcat

1. **Запустите Android Studio**
2. **Откройте проект** `client/android`
3. **Внизу экрана** найдите вкладку **"Logcat"**
   - Если вкладки не видно, нажмите **View → Tool Windows → Logcat**
   - Или используйте горячую клавишу: **Alt + 6** (Windows/Linux) или **Cmd + 6** (Mac)

## Настройка фильтров

### Фильтр по приложению:
1. В выпадающем списке **"Show only selected application"** выберите ваше приложение
2. Или выберите устройство/эмулятор в списке устройств

### Фильтр по тегам:
В поле поиска введите:
- `chromium` - для просмотра ошибок веб-вью
- `Capacitor` - для логов Capacitor
- `console` - для console.log из JavaScript
- `ERROR` - только ошибки
- `API` - если вы добавили теги в код

### Фильтр по уровню:
Выберите уровень логирования:
- **Verbose** - все логи
- **Debug** - отладочная информация
- **Info** - информационные сообщения
- **Warn** - предупреждения
- **Error** - только ошибки

## Что искать в логах

### Ошибки API запросов:
Ищите строки содержащие:
- `API request failed`
- `fetch`
- `NetworkError`
- `CORS`
- `timeout`
- `Failed to fetch`

### JavaScript ошибки:
- `Uncaught`
- `TypeError`
- `ReferenceError`
- `SyntaxError`

### Capacitor ошибки:
- `Capacitor`
- `Native`
- `Plugin`

## Полезные команды для фильтрации

В поле поиска Logcat можно использовать регулярные выражения:
- `API|Error|Failed` - найти все строки с этими словами
- `^E/` - только ошибки (Error level)

## Просмотр логов в реальном времени

1. **Подключите телефон** через USB
2. **Включите отладку по USB** на телефоне:
   - Настройки → О телефоне → Нажмите 7 раз на "Номер сборки"
   - Настройки → Для разработчиков → Отладка по USB
3. **Запустите приложение** на телефоне
4. **В Logcat** выберите ваше устройство и приложение
5. Логи будут появляться в реальном времени

## Сохранение логов

1. Нажмите правой кнопкой в окне Logcat
2. Выберите **"Save Logcat to File"**
3. Выберите место для сохранения

## Очистка логов

Нажмите кнопку **"Clear Logcat"** (иконка корзины) для очистки текущих логов

## Альтернативный способ через ADB

Если Android Studio не открыт, можно использовать командную строку:

```bash
adb logcat
```

С фильтрацией:
```bash
adb logcat | grep -i "api\|error\|capacitor"
```

Очистка логов:
```bash
adb logcat -c
```

## Отладка конкретной проблемы

Если приложение зависает на загрузке:

1. Откройте Logcat
2. Очистите логи (Clear)
3. Откройте приложение на телефоне
4. Дождитесь проблемы
5. Остановите запись (если нужно)
6. Ищите ошибки красным цветом
7. Копируйте полный стек ошибки для анализа
